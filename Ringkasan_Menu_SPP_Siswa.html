Ringkasan Menu SPP Siswa

Cakupan dokumen: role Admin SPP, role Siswa, dan role Admin untuk input data siswa.

1. Gambaran Umum

Modul SPP siswa di sistem ini terdiri dari dua sisi utama. Sisi pertama adalah modul pengelolaan oleh petugas sekolah, terutama untuk membuat tagihan, memantau transaksi, dan mengatur metode pembayaran. Sisi kedua adalah modul siswa untuk melihat tagihan, mencetak billing, menerima notifikasi, dan berkomunikasi dengan admin sekolah.

Selain itu, ada modul Data Siswa yang menjadi kebutuhan dasar karena data siswa dipakai untuk membuat tagihan, menampilkan akun siswa, dan menghubungkan akses siswa ke sekolah yang benar.

3. Role Admin SPP

Menu yang tersedia

  • Dashboard Monitoring: menampilkan total siswa, total tagihan, tagihan lunas, tagihan belum lunas, total transaksi, nominal tagihan, nominal terbayar, pengaturan aktif, overdue, coverage, paid ratio, outstanding, tagihan terbaru, dan transaksi terbaru.
  • Tagihan: membuat tagihan satuan, membuat tagihan massal, import tagihan, unduh template import, filter tagihan, dan hapus tagihan yang belum memiliki transaksi.
  • Transaksi: memantau daftar transaksi pembayaran, status pembayaran, nomor VA, dan pencarian transaksi per siswa atau nomor transaksi.
  • Laporan: melihat rekap per siswa, jumlah kelas, total tagihan, nominal terbayar, persentase pembayaran, dan ringkasan distribusi siswa per kelas.
  • Pengaturan: mengatur tahun ajaran, provider pembayaran, status aktif/nonaktif, dan catatan pengaturan.

Kebutuhan dan aturan

  • Akun harus memiliki role admin_spp dan terhubung ke satu madrasah_id.
  • Cakupan akses hanya untuk madrasah milik admin_spp tersebut, bukan seluruh sekolah.
  • Data siswa wajib sudah tersedia lebih dulu, karena tagihan dibuat berdasarkan siswa yang tersimpan di modul data siswa.
  • Untuk membuat tagihan, Admin SPP wajib memakai pengaturan aktif dengan provider BNI Virtual Account.
  • Admin SPP tidak bisa membuat tagihan jika belum ada pengaturan aktif BNI VA pada madrasahnya.
  • Pada menu pengaturan, role Admin SPP hanya diarahkan ke provider BNI Virtual Account, bukan manual.
  • Jika sekolah ingin siswa bisa mencetak billing dan menerima nomor VA, maka konfigurasi BNI VA di level aplikasi juga harus aktif.

Kebutuhan data saat membuat tagihan

Jenis Proses Data yang Dibutuhkan
Tagihan satuan Madrasah, siswa, pengaturan aktif BNI VA, periode, jatuh tempo, nominal, catatan
Tagihan massal Madrasah, pengaturan aktif BNI VA, filter jurusan/kelas jika perlu, periode, jatuh tempo, nominal, catatan
Import tagihan Madrasah, pengaturan aktif BNI VA, file xlsx/xls/csv, kolom wajib: nis, jenis_tagihan = SPP, periode, jatuh_tempo, nominal
Catatan penting: modul ini dikunci untuk tagihan jenis SPP. Jadi pembuatan tagihan dalam menu ini memang fokus ke SPP siswa, bukan jenis tagihan lain.

4. Role Siswa

Menu yang tersedia

  • Dashboard: melihat ringkasan persentase pembayaran, total tagihan, total terbayar, update terbaru, grafik lunas vs belum lunas, reminder H-3, dan tagihan terbaru.
  • Tagihan: melihat seluruh invoice/tagihan siswa, status pembayaran, detail tagihan, dan akses ke billing jika menggunakan BNI VA.
  • Pembayaran: melihat tagihan aktif, nominal, jatuh tempo, status tagihan, serta proses cetak billing dan nomor Virtual Account BNI.
  • Riwayat Pembayaran: melihat transaksi pembayaran terdahulu serta filter status dan bulan.
  • Notifikasi: melihat reminder jatuh tempo dan notifikasi pembayaran.
  • Chat Admin: mengirim pesan ke admin sekolah terkait pembayaran.
  • Profil: melihat data utama akun siswa, seperti NIS, kelas, jurusan, nomor HP, alamat, dan sekolah.
  • Halaman tambahan dari alur transaksi: Detail Tagihan, Billing, dan Bukti Pembayaran.

Kebutuhan dan aturan

  • Akun harus memiliki role siswa.
  • Data siswa harus sudah ada pada tabel siswa dan harus cocok dengan email akun serta madrasah, karena sistem mencari data siswa berdasarkan email dan madrasah.
  • Siswa baru bisa melihat isi modul pembayaran jika tagihan sudah dibuat oleh pihak sekolah.
  • Fitur BNI Virtual Account hanya berjalan jika tagihan memakai provider bni_va dan pengaturan BNI VA global aplikasi sedang aktif.
  • Nomor VA diterbitkan atau dipakai ulang saat siswa membuka/cetak billing tagihan yang aktif.
  • Fitur chat membutuhkan adanya akun admin/pengurus pada madrasah terkait. Jika tidak ada, sistem akan mencoba memakai akun super admin sebagai fallback.
  • Reminder otomatis akan muncul bila tagihan belum lunas dan tanggal jatuh tempo berada dalam rentang H-3 sampai hari H.

Kebutuhan minimum agar menu siswa berjalan baik

Fitur Siswa Kebutuhan
Dashboard dan Tagihan Data siswa aktif dan sudah memiliki tagihan
Pembayaran BNI VA Tagihan memakai provider bni_va, pengaturan aktif, dan BNI VA global aplikasi aktif
Riwayat dan Bukti Sudah ada transaksi pembayaran yang tercatat
Chat Admin Tersedia akun admin/pengurus atau fallback super admin
Notifikasi reminder Tagihan belum lunas dan memiliki tanggal jatuh tempo

5. Role Admin untuk Input Data Siswa

Menu yang tersedia

  • Data Siswa: daftar siswa lengkap dengan filter kelas, jurusan, pencarian, statistik total siswa, akun aktif, jumlah madrasah, dan jumlah kelas.
  • Tambah Siswa: input manual data siswa melalui form lengkap.
  • Edit Siswa: memperbarui data identitas, akademik, kontak, dan data orang tua/wali.
  • Hapus Siswa: menghapus data siswa.
  • Import Siswa: unggah file Excel/CSV untuk tambah atau update data siswa secara massal.
  • Template Siswa: mengunduh format template resmi sebelum import.

Kebutuhan input manual

  • Field minimal yang wajib diisi adalah NIS, nama peserta didik, kelas, dan madrasah.
  • Email siswa bersifat opsional. Jika kosong, sistem akan membuat email internal otomatis.
  • Password default akun siswa adalah NIS.
  • Role admin hanya bekerja pada madrasah miliknya sendiri, sehingga pilihan madrasah dikunci pada form.

Kebutuhan import data siswa

Komponen Kebutuhan
Format file .xlsx, .xls, atau .csv
Kolom wajib nis, kelas, nama_peserta_didik
Madrasah Dipilih pada form import, tidak wajib ditulis di file
Duplikasi data Jika NIS, NISN, atau email siswa sudah ada, sistem akan memperbarui data yang ada
Password akun baru Otomatis dibuat dari NIS

Data yang dibutuhkan saat upload data siswa

Saat upload/import data siswa, sistem membaca file template dengan susunan kolom tertentu. Tidak semua kolom wajib diisi, tetapi semakin lengkap datanya maka semakin baik untuk kebutuhan tagihan, akun siswa, dan informasi orang tua/wali.

Kelompok Data Kolom Keterangan
Wajib nis, nama_peserta_didik, kelas Ini adalah data minimum agar baris siswa bisa diproses saat upload.
Identitas siswa nisn, nik, no_kk, jenis_kelamin, tempat_lahir, tanggal_lahir, agama Dipakai untuk melengkapi profil siswa dan kebutuhan administrasi.
Akademik jurusan, tahun_masuk Mendukung pengelompokan data siswa dan pembuatan tagihan per kelas/jurusan.
Kontak siswa no_hp_siswa, email_siswa Email siswa opsional. Jika kosong, sistem membuat email internal otomatis.
Alamat alamat, dusun, kelurahan, kecamatan, kode_pos Jika alamat utama kosong, sistem tetap bisa membentuk alamat dari data wilayah yang tersedia.
Data ayah nama_ayah, pendidikan_ayah, pekerjaan_ayah, penghasilan_ayah Untuk data keluarga siswa.
Data ibu nama_ibu, pendidikan_ibu, pekerjaan_ibu, penghasilan_ibu Untuk data keluarga siswa.
Data wali nama_wali, pendidikan_wali, pekerjaan_wali, penghasilan_wali Diisi jika siswa memiliki wali yang perlu dicatat.
Kontak orang tua/wali nama_orang_tua_wali, no_hp_orang_tua_wali, email_orang_tua_wali Jika nama_orang_tua_wali kosong, sistem dapat mengambil dari nama wali, ayah, atau ibu yang tersedia.
Data tambahan jenis_tinggal, alat_transportasi Opsional untuk melengkapi informasi siswa.
Poin penting upload: kolom madrasah tidak wajib ada di file karena madrasah dipilih dari form upload. Jika NIS, NISN, atau email siswa sudah pernah ada, sistem akan memperbarui data yang lama, bukan menambah data baru.

Kolom yang didukung pada template siswa

Template import siswa mendukung data identitas, akademik, alamat, kontak, serta data ayah, ibu, dan wali. Contoh kolomnya meliputi: nis, nisn, nik, no_kk, nama_peserta_didik, jenis_kelamin, tempat_lahir, tanggal_lahir, agama, kelas, jurusan, tahun_masuk, alamat, no_hp_siswa, email_siswa, serta data orang tua/wali.

Catatan tambahan: pada implementasi saat ini, role admin_spp tidak mendapat tombol tambah siswa, import, atau template. Namun data siswa yang sudah ada tetap menjadi fondasi utama agar modul SPP siswa dapat berjalan dengan benar.

6. Kesimpulan

Secara alur kerja, Admin menyiapkan data siswa lebih dahulu, lalu Admin SPP membuat pengaturan pembayaran dan tagihan SPP, kemudian Siswa memakai menu siswa untuk melihat tagihan, mencetak billing, membayar, dan memantau status pembayaran. Jadi kualitas data siswa dan kelengkapan pengaturan pembayaran menjadi dua kebutuhan paling penting agar menu SPP siswa berjalan lancar.
